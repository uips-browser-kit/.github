# Contributing to uips-browser-kit

## Contribution model

This organization uses a tiered contribution model.

| Repository | Who can write | Contribution agreement |
|---|---|---|
| library | maintainers | Yes |
| rules | maintainers | Yes |
| snippets | maintainers | Yes |
| testproject | contributors | No |
| testdata | contributors | No |
| testharness-webapps | contributors | No |
| docs | contributors, docs | No |
| playground | contributors | No |
| exploration | contributors | No |

## Where to start

| What | Where |
|---|---|
| Bug reports, feature requests | [library](https://github.com/uips-browser-kit/library) issues |
| Workflow Analyzer rule ideas | [rules](https://github.com/uips-browser-kit/rules) issues |
| Documentation | [docs](https://github.com/uips-browser-kit/docs) |
| Experiments and prototypes | [playground](https://github.com/uips-browser-kit/playground) |
| Discussions and questions | [discussions](https://github.com/uips-browser-kit/discussions) |

## Core repositories — library, rules, snippets

Changes to these repositories require a contribution agreement.
Maintainers assess scope proposals before implementation begins.

1. Start a discussion or open an issue to propose the change.
2. A maintainer will confirm scope and assign or implement the work.

Do not open unsolicited pull requests against these repositories.

## Contribution repositories — testproject, testdata, testharness-webapps, docs

Team members (contributors, docs) may open pull requests directly.
See the repository README for what belongs in each repo.

## Open repositories — playground, exploration

Contributors may push directly — no PR required.
Code here is experimental and may not be reviewed or maintained.

## Code of Conduct

Follow the [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.
